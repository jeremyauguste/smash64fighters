❯ Can you tell me about CRA before we proceed? 

● Sure! Create React App (CRA) is an official tool from Meta (formerly Facebook)
   that scaffolds a new React project with zero configuration. Running npx 
  create-react-app my-app gives you a fully working setup out of the box
  including:

  - Webpack — bundles your JS, CSS, and assets
  - Babel — transpiles modern JavaScript and JSX
  - ESLint — basic linting rules
  - Jest — test runner pre-configured
  - Dev server — npm start launches a local server with hot reloading

  The idea is that you never have to touch config files — Webpack, Babel, etc.
  are all hidden behind react-scripts. If you ever need to customize them, you
  run npm run eject, which permanently exposes all the config files but can't be
   undone.

  The tradeoff: CRA is convenient but opinionated and slow compared to modern
  alternatives. It's largely considered outdated now — the official React docs
  no longer recommend it. The current go-to choices are:

  ┌─────────┬────────────────────────────────────────┐
  │  Tool   │             Why choose it              │
  ├─────────┼────────────────────────────────────────┤
  │ Vite    │ Much faster dev server and builds      │
  ├─────────┼────────────────────────────────────────┤
  │ Next.js │ Full-stack React with SSR/SSG built in │
  ├─────────┼────────────────────────────────────────┤
  │ Remix   │ Full-stack, focused on web standards   │
  └─────────┴────────────────────────────────────────┘

  For this project it's totally fine — it's a small client-side app and CRA will
   work without any issues. Just good context to have if you build more React
  projects in the future.
